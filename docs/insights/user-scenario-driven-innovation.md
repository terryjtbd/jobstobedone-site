# JTBD：用户场景驱动的创新

> 作者原创｜Terry

上午 8 点 30 分，我在上海南京路步行街一家五星级酒店的 38 层会议室，等待即将到来的同学们——来自这家酒店连锁集团的总部管理者、酒店业主代表和各地酒店总经理。大家要讨论冲刺 Q4 的业绩目标，而我准备介绍一个实现业务创新突破的工具：Jobs-To-Be-Done（JTBD，待完成的工作）。

## 一念之差

设想你在咖啡厅等待素未谋面的网友，突然发现手机电量不足。你想到“要是有个充电宝就好了”，结果错过见面，还被拉黑。

如果只把客户需求理解为“需要一个充电宝”，创业者很可能进入销售充电宝的红海；如果进一步追问客户要完成的工作是“在关键时刻为手机充电”，则可能想到共享充电宝服务，打开完全不同的业务方向。

不是看客户需要什么就提供什么，而是先问：**客户到底有一项什么工作要完成？**

## 营销短视症

哈佛商学院教授泰德·莱维特说：“人们不是想买一个四分之一英寸的钻头，而是需要一个四分之一英寸的孔。”

铁路业的困境也不是运输需求消失，而是管理者把自己定义为“铁路生意”，而不是“运输生意”。当企业以产品而不是以客户要取得的进步来定义自己，替代方案就会从视野之外出现。

## 奶昔的故事

一家快餐连锁店想提高奶昔销量，先按顾客特征细分、调整口味和分量，却没有达到预期。研究人员后来发现，将近一半奶昔在清晨售出，于是采访这些顾客：他们雇佣奶昔要完成什么工作？

清晨通勤者面对漫长、无聊的驾驶过程；他们想吃点东西打发时间，又希望到上午十点前有饱腹感；开车时还只能腾出一只手。三明治会弄脏手和方向盘，香蕉又吃得太快。奶昔恰好更适合完成这组任务。

一旦理解产品被用于完成什么任务，提升销量的思路才会变得清晰。

## JTBD 的应用

JTBD 给创新者提供的视角是：发现一个未被满足的任务，并提供可行方案。客户“雇佣”的不是产品本身，而是在具体场景中帮助自己取得进步的方案。

酒店送餐机器人的使用率一度不高，但服务人员发现住店儿童喜欢与机器人互动，于是围绕“召唤机器人”设计亲子活动，吸引了亲子客群。问题不再只是如何让机器人与其他设备竞争，而是如何理解并服务不同人群在不同场景中的待完成任务。

> JTBD 是一种用户场景驱动的创新。基于 JTBD，创新业务不再只是虚无缥缈、前途莫测的尝试。
