# Layer 2: 产品市场契合

## PMF 的 JTBD 诠释

Product-Market Fit（产品市场契合）是创业领域最常被提及的概念，但也是最容易被抽象化的。

我们用 JTBD 框架，把 PMF **具象化**：

```
产品市场契合 = 你的解决方案 × 用户的 Job
```

## 三个关键维度

### 1. Job 覆盖深度

你的产品解决了用户多少比例的待办任务？

| 层级 | 描述 |
|-----|------|
| 表层 | 只解决功能性任务 |
| 中层 | 同时覆盖情感性任务 |
| 深层 | 功能性 + 情感性 + 社会性任务全覆盖 |

### 2. 解决方案匹配度

你的方案是用户完成该 Job 的**最佳方式**吗？

评估标准：
- 效率：比现有方案更快吗？
- 效果：比现有方案更好吗？
- 体验：比现有方案更愉悦吗？

### 3. 替代方案可达性

用户切换到你有**多难**？

- 学习成本
- 迁移成本
- 信任成本

## 与价值主张画布的呼应

亚历山大·奥斯特瓦德（Alexander Osterwalder）的价值主张画布，与我们的 JTBD-PMF 框架高度一致：

| 价值主张画布 | JTBD-PMF 框架 |
|------------|--------------|
| 客户任务（Customer Jobs） | 待办任务识别 |
| 客户痛点（Pains） | 现有方案完成 Job 的障碍 |
| 客户收益（Gains） | 理想结果（Desired Outcomes） |
| 价值主张 | 解决方案 × Job 的契合描述 |

## 诊断工具：PMF 健康度检查表

- [ ] 你能清晰说出目标用户的核心 Job 是什么吗？
- [ ] 用户是否在没有外力推动的情况下持续使用？
- [ ] 用户是否愿意推荐给其他人？
- [ ] 你的核心指标是否在自然增长？
- [ ] 用户是否愿意为产品付费（或表现出强烈的付费意愿）？
