# 方法论框架

## 四层架构：从洞察到执行

我们的方法论是一个完整的四层架构，从用户洞察的原点出发，逐步构建战略并落地执行。

```
Layer 1  客户洞察              JTBD（原点）
         └─ 用户雇佣你的产品/服务完成什么待办任务？

Layer 2  产品-市场契合          JTBD × Product-Market Fit
         └─ 你的解决方案 × 用户的 Job = PMF 的具象化
         └─ 呼应 Osterwalder 价值主张画布

Layer 3  战略设计              Playing to Win（Lafley & Martin）
         ├─ Winning Aspiration  你要赢什么？（愿景）
         ├─ Where to Play       目标市场选择
         ├─ How to Win          如何赢得市场
         └─ Capabilities        需要什么能力？

Layer 4  执行落地              IBM 业务领先模型（BLM）
         ├─ 战略举措
         ├─ 行动计划
         └─ 组织能力承接
```

## 核心洞察

市面上大多数 JTBD 咨询，停在了 **Layer 1**——告诉你用户有什么 Job，然后就没然后了。

问题在于：

> 「发现了 Job，和「用 Job 赚到钱」之间，有一条很宽的河。」

我们的差异点是**把 JTBD 从洞察工具升级为战略操作系统**：

- **传统 JTBD 顾问**：帮你找到用户的待办任务
- **我们**：从待办任务出发，帮你定义赢什么、在哪赢、怎么赢，一直推到组织能力和行动计划

这就是 JTBD + 战略咨询的一体化。不是两个业务的拼凑，而是 **JTBD 本身就应该是战略的起点**。
