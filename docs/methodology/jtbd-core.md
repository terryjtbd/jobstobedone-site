# Layer 1: 客户洞察 - JTBD原点

## 什么是 JTBD？

JTBD（Jobs to be Done）即「待办任务理论」，由哈佛大学教授克莱顿·克里斯坦森（Clayton Christensen）提出。

**核心观点**：

> 用户购买产品或服务，不是为了产品本身，而是为了**完成某项特定的任务或解决某个特定的问题**。

### 经典比喻

> 「人们其实不想买一个四分之一英寸的钻头。他们只想要一个四分之一英寸的洞！」
> —— 西奥多·莱维特

## JTBD 的核心要素

### 1. 功能性任务（Functional Job）

用户想要完成的实际任务：
- 用更短的时间完成报告
- 降低运营成本
- 提升客户满意度

### 2. 情感性任务（Emotional Job）

用户想要感受到的情绪：
- 感觉自己很专业
- 获得掌控感
- 减少焦虑

### 3. 社会性任务（Social Job）

用户想要展现的形象：
- 被视为创新者
- 获得同事认可
- 建立专业声誉

## 为什么 JTBD 比传统用户研究更有效？

| 传统方法 | JTBD 方法 |
|---------|----------|
| 关注用户是谁（人口统计） | 关注用户想完成什么 |
| 问「你需要什么功能」 | 问「你在什么情况下会雇佣这个产品」 |
| 基于现有产品优化 | 基于任务重新定义解决方案 |
| 容易被用户表面的需求误导 | 挖掘深层的、未被满足的任务 |

## JTBD 访谈的关键问题

1. **触发点**：是什么让你开始寻找解决方案？
2. **旧方案**：之前你是怎么解决这个问题的？
3. **切换点**：是什么让你决定换一个新的方案？
4. **购买过程**：你是怎么找到并选择这个产品的？
5. **使用体验**：实际使用中和预期有什么不同？

## 从洞察到行动

JTBD 洞察的真正价值，在于它能**直接指导产品决策**：

- 识别未被充分满足的 Job → 产品创新机会
- 理解用户如何评估成功 → 定义关键指标
- 发现用户切换的障碍 → 优化转化路径
